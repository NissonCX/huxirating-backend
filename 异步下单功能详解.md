# 异步下单功能详解

## 整体架构

秒杀下单采用 **Redis 预扣 + RabbitMQ 异步入库** 的架构，将校验和扣减在 Redis 中原子完成，通过消息队列异步写入数据库，实现高吞吐、最终一致的秒杀系统。

```
用户请求 → Lua脚本(Redis原子校验) → RabbitMQ → Consumer异步入库
                                        ↓ 发送失败
                                    Outbox表 → 定时补偿重发
                                        
Consumer失败 → 重试队列(TTL 5s) → 重回主队列(最多3次) → 死信队列(DLQ) → 回滚Redis
```

## 核心流程

### 1. 接口层 — VoucherOrderServiceImpl.seckillVoucher()

```java
public Result seckillVoucher(Long voucherId) {
    Long userId = UserHolder.getUser().getId();
    long orderId = redisIdWorker.nextId("order");

    // 1. Lua 脚本原子校验：库存 + 一人一单 + 扣减
    Long result = stringRedisTemplate.execute(
            SECKILL_SCRIPT,
            Collections.emptyList(),
            voucherId.toString(), userId.toString()
    );
    int r = result.intValue();
    if (r != 0) {
        return Result.fail(r == 1 ? "库存不足" : "不能重复下单");
    }

    // 2. 通过 RabbitMQ 异步处理订单入库
    OrderMessage msg = new OrderMessage(orderId, userId, voucherId);
    try {
        rabbitTemplate.convertAndSend(
            RabbitMQConfig.ORDER_EXCHANGE,
            RabbitMQConfig.ORDER_ROUTING_KEY,
            JSONUtil.toJsonStr(msg)
        );
    } catch (Exception e) {
        // MQ 发送失败，写入 Outbox 表由定时任务补偿
        saveToOutbox(orderId, msg);
    }
    return Result.ok(orderId);
}
```

**要点**：
- Lua 脚本保证 Redis 中**库存校验 + 扣减 + 用户去重**的原子性
- MQ 发送失败不影响用户体验，Outbox 兜底

### 2. Lua 脚本 — seckill.lua

```lua
local voucherId = ARGV[1]
local userId = ARGV[2]

local stockKey = 'seckill:stock:' .. voucherId
local orderKey = 'seckill:order:' .. voucherId

if (tonumber(redis.call('get', stockKey)) <= 0) then
    return 1  -- 库存不足
end
if (redis.call('sismember', orderKey, userId) == 1) then
    return 2  -- 重复下单
end
redis.call('incrby', stockKey, -1)
redis.call('sadd', orderKey, userId)
return 0
```

### 3. MQ 消费者 — OrderMessageConsumer

```java
@RabbitListener(queues = RabbitMQConfig.ORDER_QUEUE)
public void handleOrder(String msgJson) {
    OrderMessage msg = JSONUtil.toBean(msgJson, OrderMessage.class);

    // 幂等校验：订单已存在则跳过
    if (voucherOrderService.getById(msg.getOrderId()) != null) {
        return;
    }

    // Redisson 分布式锁兜底
    RLock lock = redissonClient.getLock("lock:order:" + msg.getUserId());
    if (!lock.tryLock()) {
        throw new RuntimeException("获取锁失败，触发重试");
    }
    try {
        // 数据库二次校验一人一单
        int count = voucherOrderService.query()
            .eq("user_id", msg.getUserId())
            .eq("voucher_id", msg.getVoucherId())
            .count();
        if (count > 0) return;

        // 乐观锁扣减库存
        boolean success = seckillVoucherService.update()
            .setSql("stock = stock - 1")
            .eq("voucher_id", msg.getVoucherId())
            .gt("stock", 0)
            .update();
        if (!success) throw new RuntimeException("库存不足");

        // 创建订单
        VoucherOrder order = new VoucherOrder();
        order.setId(msg.getOrderId());
        order.setUserId(msg.getUserId());
        order.setVoucherId(msg.getVoucherId());
        voucherOrderService.save(order);
    } finally {
        lock.unlock();
    }
}
```

**要点**：
- **幂等**：消费前查库判断订单是否已存在
- **分布式锁**：按 userId 加锁，防止并发消费同一用户的多条消息
- **乐观锁**：`stock > 0` 条件防止超卖

### 4. 失败处理 — 重试队列 + 死信队列

#### RabbitMQ 拓扑

```
order.exchange ──(order.routing.key)──→ order.queue (主消费队列)
                                              ↓ 消费失败 (nack)
                                        order.retry.queue (TTL=5s, 无消费者)
                                              ↓ TTL 过期
                                        order.exchange (死信路由回主队列)
                                              ↓ 重试超过3次
                                        order.dlx.exchange → order.dlq (死信队列)
```

#### 重试机制
- 消费失败时，消息的 `x-retry-count` header 递增
- 若 retryCount < 3：发送到 retry.queue（TTL 5s 后重回主队列）
- 若 retryCount >= 3：发送到死信队列

#### 死信队列处理 — DeadLetterConsumer
```java
@RabbitListener(queues = RabbitMQConfig.DEAD_LETTER_QUEUE)
public void handleDeadLetter(String msgJson) {
    OrderMessage msg = JSONUtil.toBean(msgJson, OrderMessage.class);
    // 回滚 Redis：恢复库存 + 移除用户记录
    stringRedisTemplate.opsForValue().increment(
        "seckill:stock:" + msg.getVoucherId());
    stringRedisTemplate.opsForSet().remove(
        "seckill:order:" + msg.getVoucherId(), msg.getUserId().toString());
}
```

### 5. 补偿机制 — OrderCompensationTask

```java
@Scheduled(fixedDelay = 30000)
public void compensate() {
    // 查询 Outbox 中 status=0 且 retry_count < 5 的消息
    List<MessageOutbox> list = outboxService.list(
        new LambdaQueryWrapper<MessageOutbox>()
            .eq(MessageOutbox::getStatus, 0)
            .lt(MessageOutbox::getRetryCount, 5)
    );
    for (MessageOutbox outbox : list) {
        try {
            rabbitTemplate.convertAndSend(..., outbox.getMessageBody());
            outbox.setStatus(1); // 发送成功
        } catch (Exception e) {
            outbox.setRetryCount(outbox.getRetryCount() + 1);
            if (outbox.getRetryCount() >= 5) {
                outbox.setStatus(2); // 彻底失败
            }
        }
        outboxService.updateById(outbox);
    }
}
```

### 6. Outbox 表结构

```sql
CREATE TABLE tb_message_outbox (
    id           BIGINT AUTO_INCREMENT PRIMARY KEY,
    order_id     BIGINT NOT NULL,
    message_body TEXT NOT NULL,
    status       TINYINT DEFAULT 0 COMMENT '0-待发送 1-已发送 2-失败',
    retry_count  INT DEFAULT 0,
    create_time  DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time  DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_status_retry (status, retry_count)
) COMMENT '消息发件箱';
```

## 一致性保障总结

| 层级 | 机制 | 说明 |
|------|------|------|
| Redis 层 | Lua 脚本 | 库存校验+扣减+去重的原子性 |
| MQ 投递 | Publisher Confirm | 确认消息到达交换机 |
| MQ 投递失败 | Outbox + 定时补偿 | 保证消息最终被投递 |
| 消费幂等 | 数据库查询去重 | 防止重复创建订单 |
| 消费失败 | 重试队列 (TTL + DLX) | 延时重试最多 3 次 |
| 彻底失败 | 死信队列 | 回滚 Redis 数据，恢复一致 |
| 并发控制 | Redisson 分布式锁 | 兜底防止数据库并发写入 |
| 超卖防护 | 乐观锁 `stock > 0` | 数据库层最终防线 |
