## 报告漏洞

如果您发现安全漏洞，请**不要**在公开的 Issue 中报告。

请通过以下方式报告：

1. 发送邮件至 [nisson@example.com](mailto:nisson@example.com)
2. 在邮件标题中包含 `[SECURITY]` 前缀
3. 提供详细的漏洞描述，包括复现步骤

## 响应时间

我们将在收到报告后的 **48 小时内** 确认收到，并在 **7 天内** 提供修复方案或更新时间表。

## 安全最佳实践

### 数据库

- 使用环境变量存储敏感凭证
- 生产环境请修改默认密码
- 启用 SSL 连接

### Redis

- 设置强密码
- 启用 ACL 控制
- 限制外部网络访问

### RabbitMQ

- 修改默认 guest 用户密码
- 创建专用用户并分配最小权限

## 依赖项安全

我们定期更新依赖项以修复已知漏洞。建议：

